# 🗂️ File Organizer

> Bash-скрипт для автоматической сортировки файлов по расширениям с поддержкой dry-run и отката изменений.

## Описание

Лабораторная работа демонстрирует практическое применение Bash-скриптинга: работу с файловой системой, обработку аргументов командной строки, логирование, цветной вывод в терминал и механизм отката изменений.

Скрипт сканирует текущую директорию и перемещает файлы в подпапки, названные по расширению (`pdf_dir`, `jpg_dir`, `no_ext_dir` и т.д.).

---

## Возможности

- Сортировка файлов по расширению в соответствующие папки
- Режим предпросмотра (`--dry-run`) без реального перемещения файлов
- Откат последней операции (`--undo`) через журнал изменений
- Обработка коллизий имён (автоматическое переименование дубликатов)
- Цветной вывод в терминал (отключается при перенаправлении вывода)
- Подробный режим (`--verbose`) для отладки
- Поддержка скрытых файлов (`--hidden`)

---

## Требования

- Bash 3.2 или новее
- macOS или Linux
- Права на запись в текущую директорию

---

## Установка

```bash
git clone <url-репозитория>
cd <папка-проекта>
chmod +x organizer.sh
```

---

## Использование

```bash
./organizer.sh [ОПЦИИ]
```

### Опции

| Опция | Описание |
|---|---|
| `-n`, `--dry-run` | Предпросмотр: показать, что будет перемещено, без реального перемещения |
| `-v`, `--verbose` | Подробный вывод (INFO-сообщения) |
| `-h`, `--hidden` | Включить скрытые файлы (начинающиеся с `.`) |
| `--undo` | Откатить последний запуск (восстановить файлы по журналу) |
| `--help` | Показать справку |

### Примеры

```bash
# Просмотреть, что будет перемещено, не трогая файлы
./organizer.sh --dry-run

# Запустить сортировку с подробным выводом
./organizer.sh --verbose

# Сортировать включая скрытые файлы
./organizer.sh --hidden

# Откатить последнюю операцию
./organizer.sh --undo

# Комбинирование опций
./organizer.sh -n -v -h
```

## Журнал и откат

Каждое перемещение записывается в файл `.organizer_journal.log` в формате:

```
[2025-01-15 14:23:01] INFO: Будет создана/использована папка: pdf_dir
[2025-01-15 14:23:01] WARN: Коллизия имен: report.pdf -> report_1.pdf
```

Для отката используйте:

```bash
./organizer.sh --undo
```

После успешного отката журнал очищается.

> **Внимание:** откат восстанавливает файлы только из **последнего** запуска. Повторный вызов `--undo` после очистки журнала не имеет эффекта.

```

Перемещено файлов: 12
Ошибок: 0
Пропущено (системные/скрытые/скрипт): 3
```

## Технические детали

- Скрипт и файл журнала исключаются из обработки автоматически
- Скрытые файлы (`.dotfiles`) пропускаются по умолчанию
- Расширения приводятся к нижнему регистру (`JPG` → `jpg_dir`)
- Используется `find -print0` + `read -d ''` для корректной обработки пробелов в именах файлов
- Режим `set -uo pipefail` обеспечивает перехват ошибок

---

## Ограничения

- Обрабатываются только файлы в **текущей директории** (без рекурсии, `-maxdepth 1`)
- Откат не поддерживает несколько уровней истории
